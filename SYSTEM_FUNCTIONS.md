# 系统功能说明文档

## 1. 系统概览
本系统是一个集成了**OCR/大模型智能解析**、**向量语义检索**与**项目档案管理**的智能化工程物料管理平台。旨在解决传统工程资料管理中“搜索难、录入繁、复用低”的痛点，实现从资料录入到报审归档的全流程数字化。

## 2. 核心技术栈
- **前端框架**: Next.js 14 (App Router) + React
- **UI 组件库**: Ant Design + Tailwind CSS
- **后端服务**: Next.js API Routes (业务逻辑) + Python FastAPI (AI 服务)
- **数据库**: SQLite + Prisma ORM
- **AI/算法**: 
  - **OCR/VLM**: Qwen-VL-Plus (通义千问视觉大模型)
  - **向量检索**: ChromaDB + BGE-large-zh
  - **关键词检索**: BM25
  - **重排序**: RRF (Reciprocal Rank Fusion) + LLM Rerank (Qwen-32B)

## 3. 核心功能模块

### 3.1 智能资料录入 (Data Entry & Parsing)
系统通过 AI 技术极大简化了繁琐的证书录入流程：
- **多模态上传**: 支持上传 PDF 或图片格式的证书与检测报告。
- **智能解析**: 调用视觉大模型（VLM）自动提取关键元数据：
  - **厂家名称** (Manufacturer Name)
  - **物料名称** (Material Name)
  - **规格型号** (Model/Specification)
  - **有效期/报告日期** (Expiry/Report Date)
- **自动归档**:
  - **智能重命名**: 上传成功后，文件自动重命名为规范格式（如 `厂家-物料-型号-类型.pdf`）。
  - **查重机制**: 自动检测库中是否已存在同厂家、同物料、同类型的文档，避免重复录入。
  - **向量化**: 后台异步将文档内容切片并向量化，为语义搜索提供支持。

### 3.2 双引擎物料搜索 (Dual-Engine Search)
提供比传统关键词搜索更强大的检索能力：
- **混合检索**: 同时使用关键词匹配 (BM25) 和语义向量匹配 (Vector Search)。即使用户输入的词汇与文档不完全一致（如“合格证”vs“质量证明书”），也能通过语义关联搜到。
- **高级筛选**: 支持按“厂家”、“物料名称”、“规格型号”、“适用国别”及“证书类型”进行精确过滤。
- **智能高亮**: 搜索结果不仅展示文档，还能自动高亮显示文档中与查询最相关的片段和数值。
- **AI 问答 (RAG)**: (部分实现) 基于检索到的文档片段，通过大模型直接回答用户关于物料参数的提问。

### 3.3 项目档案管理 (Project Archive Management)
实现“全局库”与“项目库”的分离与联动：
- **全局资源引用**: 项目人员无需重新上传文件，可直接从全局物料库中搜索并“引用”物料到当前项目。
- **项目专属定制**:
  - **元数据覆盖**: 项目中引用的物料可拥有独立的名称和型号（通过 `parsedMeta` 存储），不影响全局库原始数据。
  - **专属文档**: 支持上传项目特有的过程文件（如“封样单”），这些文件仅在当前项目可见，不会污染全局库。
- **权限隔离**: 只有项目成员才能访问和编辑项目档案，支持管理员审核成员加入申请。

### 3.4 自动化报审导出 (Export & Reporting)
一键生成标准化的报审资料包：
- **自动生成 Excel**: 根据项目物料清单，自动提取物料名称、规格型号（优先从证书提取）、厂家名称等信息，生成《物料报审表.xlsx》。
- **资料打包**: 自动归集所有相关证书（营业执照、ISO证书、产品合格证、检测报告、封样单），按“物料-厂家”建立文件夹结构，打包为 ZIP 下载。

### 3.5 后台管理 (Admin)
- **审核机制**: 普通用户上传的全局资料默认为“待审核”状态，需管理员批准后才对所有人可见。
- **用户管理**: 简单的角色管理（管理员/普通员工）。

---

## 4. 数据流转逻辑
1. **入库**: 用户上传 -> AI 解析 -> 存入 `MasterMaterial` / `MasterDocument` (全局库)。
2. **建项**: 创建 `Project` -> 搜索全局物料 -> 创建 `ProjectArchiveItem` (引用关系)。
3. **定制**: 在项目中修改物料信息 -> 更新 `ProjectDocument` 的 `parsedMeta`。
4. **出库**: 点击导出 -> 读取 `ProjectArchiveItem` 及 `parsedMeta` -> 生成最终交付物。
